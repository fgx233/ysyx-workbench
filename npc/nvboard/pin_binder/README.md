# NVBoard 图形化引脚绑定器

不再手写 `constr/top.nxdc` —— 在浏览器里把 top 模块的端口"拉线"连到一块
虚拟 NVBoard 上，保存时自动生成约束文件。零第三方依赖（仅 Python 标准库 +
原生浏览器）。

## 启动

```bash
# 在项目根目录（nvboard_example）下运行
python3 pin_binder/server.py
```

会自动打开浏览器（默认 http://127.0.0.1:8642/）。可选参数：

| 参数 | 说明 |
|---|---|
| `--project DIR` | 项目根目录，默认是 pin_binder 的上级目录 |
| `--board FILE` | 板卡描述文件，默认 `$NVBOARD_HOME/board/N4` |
| `--top NAME` / `--nxdc PATH` | 顶层模块名 / nxdc 路径，默认从 Makefile 读取 |
| `--vsrc DIR` | Verilog 源码目录（相对项目根或绝对路径），默认 `vsrc` |
| `--port N` | 监听端口，默认 8642 |
| `--no-browser` | 不自动打开浏览器 |

顶层端口从源码目录下的 Verilog 文件自动解析（ANSI 风格端口声明），
`clk` / `rst` 由仿真主循环驱动，会自动排除。本仓库作为父项目的子目录
使用时，源码在父项目的 `../vsrc` 下，推荐用 `make link` 启动（它会把
Makefile 里的 `VSRC_DIR` 通过 `--vsrc` 传进来）。

## 使用方法

1. **先看建议**：打开后右侧会列出按名称自动匹配的连接建议（虚线显示），
   大部分情况点"全部接受"即可完成全部绑定。
2. **手动连线**：点击左侧一个端口，板上所有方向匹配且空闲的引脚会发绿光：
   - **左键**点击一个发光引脚：从该位号开始**自动连续填充**整个向量
     （如 8 位的 `sw` 点 `SW7` 即得 `SW7…SW0`）；
   - **右键**点击：**逐位指定**，适合不规则映射（如 `btn`）；
     `Shift`+左键 效果相同；
   - 点击组标题（如 `SW`、`SEG3`）：按位号对齐一键填充整组；
   - VGA 的 R/G/B 行标签也可以点击，整行对齐填充；
   - `Esc` 取消。
3. **检查与修改**：点击连线或彩色引脚查看位映射明细，可反转位序、
   重新连线或删除绑定。
4. **保存**：右上角"保存 nxdc"（或 `Ctrl+S`）写回约束文件。保存前会做
   完整校验（方向、位宽、引脚冲突），不合法的绑定根本无法在界面上做出来。
   旁边的"保存并退出"会在写回成功后**自动关闭后端服务器**，省去回终端
   按 `Ctrl+C`；关闭后页面按钮置灰，可直接关掉网页。

其他快捷键：`Ctrl+Z` 撤销、`Ctrl+Shift+Z` / `Ctrl+Y` 重做。

右下角的"top.nxdc 实时预览"始终显示当前绑定生成的文件内容，所见即所得。

## 文件结构

```
pin_binder/
├── core.py      # 解析（板卡/Verilog 端口/nxdc）、校验、建议、生成；可独立测试
├── server.py    # 本地 HTTP 服务（仅标准库），/api/state /api/preview /api/save
└── web/         # 前端单页应用（原生 JS，无构建步骤）
```

生成的 nxdc 与 NVBoard 官方 `auto_pin_bind.py` 完全兼容（已用官方脚本
验证生成结果与手写文件语义一致）。引脚顺序遵循 nxdc 约定：MSB 在前。
